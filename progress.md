# 🚀 프로젝트 작업 이력 및 온보딩 가이드 (Project History & Onboarding)

본 문서는 `Billionaire Boys Club` 프로젝트의 태동(2020년)부터 현재의 All-Weather 퀀트 봇(2026년)으로 진화하기까지의 모든 작업 과정과 아키텍처 변경 내역을 기록한 문서입니다. 
새롭게 프로젝트에 참여하는 팀원이나 유지보수 담당자가 프로젝트의 발전 맥락을 빠르게 파악할 수 있도록 작성되었습니다.

## 📜 1. 프로젝트의 기원 (Phase 1: 2020년)
초창기 프로젝트는 파이썬을 이용한 단순 스크래핑과 키움증권 API 연동 가능성을 탐색하는 토이 프로젝트 성격이 강했습니다.

- **2020-11-22 ~ 11-24:** 파이썬을 이용한 키움 API 접근법 타당성 검토 및 강남역 1차 오프라인 모임 (조건식 확인 및 API 사용 확정)
- **2020-11-25:** 동전주/우선주 필터링 로직 구현 및 네이버 뉴스 크롤링(`Selenium` 기반) 로직 추가 (by cham)
- **2020-11-27 ~ 11-30:** 코스닥 전체 종목/일봉 데이터 추출 로직 추가, 2차 오프라인 모임 (스케줄링 로직 도입)
- **2020-12-07:** 텔레그램 봇 푸시 메시징 최초 연동 및 1분봉 기반 거래량 조건 세분화.

> **⚠️ 한계점 파악:** 1분봉 단위의 스캘핑 로직은 슬리피지(Slippage)와 거래 수수료에 너무 취약했으며, 수학적인 백테스트 검증이 부재하여 실전 투입이 지연됨.

## 🏗️ 2. 퀀트 아키텍처 개편 및 백테스트 도입 (Phase 2: 2026년 7월)
1분봉 스캘핑을 버리고, 일봉(Daily) 기반의 추세 추종 전략을 채택. 코드베이스를 퀀트 펀드 수준의 모듈형 아키텍처로 전면 개편.

- **전략 명세서 분리:** 매매 조건을 하드코딩하지 않고 `trading_conditions.md`로 분리하여 관리.
- **백테스트 엔진(`backtest/engine.py`, `simulator.py`) 구축:** 과거 1~3년 치 주가 데이터(`FinanceDataReader` 연동)를 바탕으로 봇의 수익률을 시뮬레이션할 수 있는 프레임워크 완성.
- **파라미터 최적화(`optimizer.py`):** Grid Search 방식을 통해 최고의 손익비와 승률을 내는 익절(+15%) 및 손절(-1%) 라인 발굴.
- **데일리 봇(`run_daily.py`):** 장 전/후로 포트폴리오를 업데이트하고 마크다운 브리핑을 이메일로 자동 송신하는 기능 도입.

## 🛡️ 3. All-Weather 헷징 및 타점 레이더 장착 (Phase 3: 2026년 7월 8일 오전)
시장이 폭락할 때 손만 빨고 있는 문제점(Drawdown)을 해결하고, 장중 단기 타점을 놓치지 않기 위한 2가지 메이저 업데이트 진행.

- **[방어] 인버스 헷징 모듈(`index_strategy.py`) 추가:**
  - 코스피 60일선 붕괴 및 RSI 역회전 시, KODEX 인버스를 대거 매수하여 하락장에서도 계좌를 방어하는 기관급 롱숏(Long-Short) 혼합 포트폴리오 완성.
  - 과거 폭락장 시뮬레이션을 통해 인버스 헷징 시 기댓값이 크게 상승함을 검증.

- **[공격] 1분 주기 장중 타점 레이더(`start_radar_daemon.py`, `radar_alert.py`) 구축:**
  - 하루 한두 번만 스캔하던 기존 데일리 봇의 단점 보완.
  - 평일 장중(09:00~15:30) 내내 1분 간격으로 인버스와 테마주 대장주의 거래량을 백그라운드에서 무한 감시.
  - 완벽한 타점 포착 시 **네이버 이메일**로 종목명, 매수가, 익절/손절가를 즉각 푸시 알림 전송.
  - `alert_history.json`을 도입하여 하루 1회 중복 발송(Spam) 방지.

- **[보안] 이메일 자격증명 깃허브 격리:**
  - `.gitignore`와 `config.json`을 분리하여 네이버 아이디/앱 비밀번호가 퍼블릭 저장소에 노출되지 않도록 강력한 보안 조치 적용. (과거 유출 사고 인지 및 Force Push 롤백/복구 완료)

## ⚡ 4. 실시간 가상 매매 엔진 및 전략 교체 (Phase 4: 2026년 7월 8일 오후)
단순 알림 봇에서 **실시간 자동 모의투자 엔진**으로 대대적인 업그레이드 진행.

- **[코어] 실시간 가상 매매 엔진 (`radar_alert.py` 리팩토링):**
  - 타점 포착 시 알림만 보내는 것이 아니라, 가상 현금(`portfolio.json`) 50만 원을 차감하고 즉시 매수 체결. 보유 종목은 매 1분마다 현재가를 체크하여 익절/손절 시 자동 매도 체결.
  - 매수/매도 이벤트 발생 시 **실시간 이메일 알림** 전송 (익절 완료, 손절 완료 등).

- **[리포팅] 데일리 브리핑 리포트 (`run_daily.py` 리팩토링):**
  - 장 마감 후(15:30) 하루 동안 레이더 봇이 수행한 모든 매매 기록을 수집하여 **마크다운 일지**로 생성.
  - 3일 보유 타임스탑 강제 청산 로직 담당.
  - 데일리 리포트를 이메일로 자동 발송.

- **[공유] 포트폴리오 매니저 분리 (`portfolio_manager.py` 신규):**
  - `load_portfolio()` / `save_portfolio()` 함수를 독립 모듈로 분리하여 레이더와 데일리 봇이 동일한 장부를 안전하게 공유.

- **[전략 교체] 옵션 B 단기 타격 전략 적용:**
  - 기존 익절 +15% / 손절 -1% (승률 10%) → **익절 +7% / 손절 -2% (승률 16.7%)**로 변경.
  - 회원님의 투자 성향에 맞추어 심리적 안정성과 승률을 최우선으로 하는 전략으로 교체.

- **[테마 확장] 감시 종목 대폭 확대:**
  - 기존 3대장(조선/원전/방산)에 추가로 **반도체**, **피지컬 AI**, **AI 에너지 인프라(전력설비)** 테마를 활성화.
  - 회원님 전용 개별 관심 종목 바구니(`themes/user_picks.csv`) 도입 → **이마트** 등록.
  - 동진쎄미켐 종목코드 오류(005960→005290) 긴급 수정.

## 🧠 5. 매크로 공포 점수(Fear Score) 기반 인버스 고도화 (Phase 5: 2026년 7월 8일 오후)
기존의 단순 가격대(930~950원) 기반 인버스 매수 로직을 폐기하고, **4대 글로벌 매크로 선행지표**를 종합하는 정량적 공포 점수(Fear Score) 시스템으로 전면 교체.

- **[신규] 매크로 지표 수집 모듈 (`backtest/macro_indicators.py`):**
  - **외국인 연속 순매도 일수 & 금액** (`pykrx` 연동): 5영업일 이상 연속 순매도 AND 누적 1조원 이상 시 위험 판정
  - **VKOSPI (한국판 공포지수)** (`FinanceDataReader` 연동): 25pt 이상 시 극도의 공포 심리 판정
  - **원/달러 환율 급등** (`FinanceDataReader` 연동): 5일 이평선 돌파 AND 1,400원 이상 시 자본유출 경고
  - **미국 10년물 국채 금리** (`FinanceDataReader` 연동): 전일 대비 +0.1%p 급등 또는 4.5% 이상 시 글로벌 위험자산 경보

- **[코어] 공포 점수(Fear Score) 시스템:**
  - 위 4개 지표를 종합하여 0~4점의 정량적 점수를 산출.
  - **2점 이상:** 인버스 매수 (매크로 황색 경보)
  - **3~4점:** 인버스 강력 매수 (매크로 적색 경보)
  - **0~1점:** 인버스 매수 보류 (매크로 안정)

- **[알림] 매크로 상세 내역 이메일 포함:**
  - 인버스 매수 체결 이메일에 공포 점수와 각 지표별 상세 수치를 포함하여, 왜 지금이 인버스 매수 적기인지를 회원님이 한눈에 파악할 수 있도록 함.

## 🔬 7. 백테스트 정직성 개편 — 수익률 착시 제거 (Phase 7: 2026-07-08 전문가 리뷰)
외부 전문가 시선으로 백테스트 방법론을 감사한 결과, **보고되던 수익률(기댓값 +0.54~1.28%, 누적 291%)이
실전 수익이 아니라 백테스트 착시**임을 확인하고 이를 코드로 바로잡음. 상세 근거: [EXPERT_REVIEW.md](EXPERT_REVIEW.md).

### 🐞 발견된 3대 구조적 왜곡
1. **갭 무시 낙관 체결(최악):** `if low <= buy*(1+sl): profit = sl*100` — 저가가 손절선을 스치기만 하면
   무조건 정확히 -1%로 기록. -6% 갭하락에도 -1%로 체결됐다고 씀. `expected_profit_loss_log.md`의
   **모든 손절이 정확히 -1.00%**인 것이 이 버그의 지문. 실측: 같은 -6% 갭에서 구 로직 -2% vs 현실 -6.36%.
2. **거래비용·세금·슬리피지 0 가정:** 한국 왕복 ≈0.38%. 연 600매매 전략에서 치명적.
   실측: 승률 22%/+7%/-2% 조건조차 비용 반영 시 기댓값 -0.336%/매매 → **복리 -91%, MDD 93%, 연속손절 25회**.
3. **과최적화 + 리스크 무지:** 13종목·단일테마에서 단리합산 최댓값 1칸을 표본외 검증 없이 채택.
   MDD·Sharpe·생존자편향 전부 무시. 승률 10.58% '최적'은 소수 +15% 꼬리 승리에 과최적합된 값.

### 🔧 코드 첨삭
- **`backtest/realistic.py`(신규):** 갭 인지 체결 `resolve_exit`(시가가 손절/익절선 넘겨 시작하면 시가 체결)
  + `CostModel`(수수료·거래세·슬리피지) + `evaluate`(복리 자산곡선·MDD·Sharpe·Profit Factor·최대연속손실).
  시장데이터 없이 도는 self-test로 갭 4%p 축소·비용 기댓값 반전을 수치 증명.
- **`backtest/optimizer.py`:** 매수시그널 날짜순 **앞70% 학습 / 뒤30% 검증(walk-forward)**.
  랭킹을 **표본외 비용반영 기댓값**으로 교체, IS≫OOS면 과최적 경보. `-1%` 손절은 후보에서 제외.
- **`backtest/engine.py` / `simulator.py`:** 낙관 체결 제거 → `resolve_exit` 사용, 리포트를 비용반영+MDD/PF로.
- **`backtest/strategy.py` v1.1→v1.2:** **추세 필터 신설**(전일 종가>20일선일 때만 진입)로 "떨어지는 칼날 위
  거래량" 손절 다발의 근원 제거 + **ATR%** 컬럼 노출(변동성 기반 손절 근거).
- **문서:** 왜곡의 산물인 `optimization_results.md`·`expected_profit_loss_log.md` 상단에 폐기 배너 부착.
